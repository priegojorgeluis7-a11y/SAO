## 3. Datos personales que SAO puede tratar

Dependiendo del rol del usuario y del tipo de operación realizada, SAO puede tratar las siguientes categorías de datos:

### 3.1 Datos de identificación y contacto
- nombre completo;
- correo electrónico;
- identificador interno de usuario;
- puesto, rol o perfil operativo;
- datos de proyecto, frente, zona o alcance asignado.

### 3.2 Datos operativos y de actividad
- registros de actividades y eventos;
- asignaciones de trabajo;
- estatus operativos y de revisión;
- observaciones, comentarios y seguimiento;
- historial de cambios y trazabilidad de acciones.

### 3.3 Evidencias y anexos
- fotografías;
- videos;
- documentos adjuntos;
- metadatos asociados a la evidencia, como fecha, hora y relación con una actividad o evento.

### 3.4 Datos de ubicación y contexto de captura
Cuando la operación lo requiera, SAO puede tratar:
- coordenadas GPS o ubicación aproximada;
- datos de fecha y hora de captura;
- referencia del dispositivo o sesión utilizada para el registro.

### 3.5 Datos técnicos y de seguridad
- tokens o credenciales de sesión;
- registros de acceso, autenticación y auditoría;
- dirección IP, tipo de dispositivo, sistema operativo o información técnica similar;
- bitácoras necesarias para soporte, integridad, seguridad y prevención de incidentes.

### 3.6 Datos sensibles
SAO **no está diseñado para solicitar de forma deliberada datos personales sensibles**. Si un usuario incorpora información sensible dentro de comentarios, evidencias o documentos, dicho tratamiento será considerado incidental y deberá limitarse estrictamente al propósito operativo o de cumplimiento correspondiente.

## 4. Finalidades del tratamiento

### 4.1 Finalidades primarias
Los datos personales tratados en SAO se utilizan para:

1. autenticar usuarios y administrar accesos según rol y alcance;
2. operar flujos de trabajo, actividades, eventos, agenda y validaciones;
3. registrar evidencias de campo y su trazabilidad;
4. coordinar asignaciones, seguimiento y supervisión operativa;
5. mantener la sincronización entre dispositivos y servicios del sistema;
6. generar reportes, auditorías, bitácoras e históricos operativos;
7. atender incidencias técnicas, soporte, continuidad y seguridad del servicio;
8. cumplir obligaciones contractuales, regulatorias, laborales, administrativas o de control interno.

### 4.2 Finalidades secundarias
De manera adicional y siempre que resulte aplicable conforme a la normativa vigente, SAO podrá usar información no sensible para:

- mejorar la experiencia de uso;
- analizar desempeño operativo del sistema;
- fortalecer controles de calidad, seguridad y confiabilidad;
- elaborar métricas agregadas para planeación, mejora continua y toma de decisiones.

SAO **no comercializa datos personales** ni los utiliza para publicidad no relacionada con la operación del servicio.

## 5. Base jurídica y consentimiento

El tratamiento de datos personales en SAO se fundamenta, según corresponda, en:

- la relación contractual o laboral con el responsable del tratamiento;
- el interés legítimo para la operación, seguridad y continuidad del servicio;
- el cumplimiento de obligaciones legales o regulatorias;
- el consentimiento del titular cuando sea requerido por ley.

Cuando el dispositivo solicite permisos como **cámara, ubicación, fotos/archivos, almacenamiento o biometría**, dichos accesos se usarán únicamente para habilitar funciones propias de SAO, tales como captura de evidencias, georreferenciación, inicio de sesión seguro o resguardo local de la sesión. En dispositivos iOS, estos permisos se solicitan de manera contextual y pueden administrarse o revocarse desde la configuración del sistema.

## 6. Transferencias y compartición de datos

SAO podrá compartir datos personales únicamente en los casos necesarios y permitidos por la ley, por ejemplo con:

- proveedores de infraestructura tecnológica, nube y almacenamiento;
- prestadores de soporte, mantenimiento o monitoreo bajo obligaciones de confidencialidad;
- empresas vinculadas o clientes del mismo grupo operativo, cuando exista base legal o contractual;
- autoridades competentes, cuando una disposición legal lo requiera.

En todos los casos aplicables, se procurará que las transferencias se sujeten a medidas contractuales, técnicas y organizativas razonables para proteger la información.

---

## 7. Conservación de la información

Los datos personales serán conservados durante el tiempo que resulte necesario para:

- cumplir las finalidades descritas en esta política;
- mantener la continuidad operativa y la trazabilidad del sistema;
- atender auditorías, revisiones, aclaraciones o controversias;
- cumplir plazos legales, contractuales o regulatorios aplicables.

Una vez concluido el periodo necesario, los datos podrán ser eliminados, anonimizados, bloqueados o resguardados conforme a la normativa aplicable y a las políticas internas de retención documental del responsable.

## 8. Medidas de seguridad

SAO incorpora medidas razonables de seguridad administrativas, técnicas y organizativas para proteger la información contra daño, pérdida, alteración, destrucción, acceso o tratamiento no autorizado. Entre ellas pueden incluirse:

- comunicaciones cifradas mediante HTTPS/TLS;
- autenticación por usuario y control de acceso por roles y alcances;
- tokens de sesión y controles de autenticación;
- registros de auditoría y trazabilidad de acciones;
- almacenamiento controlado de evidencias y archivos;
- mecanismos de respaldo, monitoreo y gestión de incidentes.

No obstante, ningún sistema es absolutamente infalible, por lo que el responsable implementará esfuerzos razonables y de mejora continua para reducir riesgos de seguridad y privacidad.

## 9. Uso de almacenamiento local, sesión y permisos del dispositivo

Para funcionar correctamente, SAO puede almacenar información mínima en el dispositivo, por ejemplo:

- preferencias de usuario;
- información de sesión y autenticación;
- datos temporales para operación offline-first;
- elementos de sincronización pendientes;
- configuraciones locales necesarias para el servicio.

Asimismo, la aplicación puede solicitar permisos como:

- **Cámara**: para capturar fotografías o video de evidencias;
- **Ubicación**: para registrar coordenadas cuando el flujo operativo lo requiera;
- **Fotos, archivos o almacenamiento**: para adjuntar, cargar o consultar documentos y evidencias guardadas en el dispositivo;
- **Biometría o PIN**: para reforzar el acceso local cuando la plataforma o configuración lo permitan, incluyendo mecanismos como **Face ID** o **Touch ID** en iOS cuando estén habilitados.

En iOS, SAO solicita únicamente los permisos necesarios para la función que el usuario ejecuta en ese momento y no accede a contenidos no relacionados con la operación del servicio.

Estos permisos no se utilizan con fines publicitarios ni de perfilamiento comercial ajenos a la operación de SAO.
